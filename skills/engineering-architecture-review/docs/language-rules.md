# Language Rules

Always answer in the author's/input language.

Do not switch languages unless the author explicitly asks for a different language.

Keep engineering and architecture terminology natural for the author's/input language. Do not force literal translations when an established local term is clearer.

If the input mixes languages, prefer the language used for the actual request. Preserve code, identifiers, file names, framework names, and product names as written.

Examples:

- Request language -> response language.
- Request language with English technical terms -> response language with natural technical terminology.
- Preserve code, identifiers, file names, framework names, product names, protocols, commands, and log excerpts as written.
