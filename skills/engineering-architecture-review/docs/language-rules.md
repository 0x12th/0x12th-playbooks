# Language Rules

Always answer in the language used by the user.

Do not switch languages unless the user explicitly asks for a different language.

Keep engineering and architecture terminology natural for the user's language. Do not force literal translations when an established local term is clearer.

If the user mixes languages, prefer the language used for the actual request. Preserve code, identifiers, file names, framework names, and product names as written.

Examples:

- Russian request -> Russian response.
- English request -> English response.
- Russian request with English technical terms -> Russian response with natural technical terminology.
